What is the different of var, let and const?

The scope of a var variable is functional scope. The scope of a let variable is block scope. The scope of a const variable is block scope. It can be updated and re-declared into the scope.

Tell us the use cases of null and undefined

Here are simple use cases for `null` and `undefined` in JavaScript:
1. **null**:
- It is explicitly assigned to indicate that a variable or object intentionally has no value or no reference. - Often used to reset or clear a variable's value. - Used when you want to represent the absence of an object or value. 2. **undefined**:
- It indicates that a variable has been declared but hasn't been assigned any value yet. - It's the default value for function parameters that are not provided. - Can occur as the result of an expression or operation that doesn't return a value.
In summary, `null` is typically used when you want to explicitly state that a variable is intentionally empty, while `undefined` is more often associated with the default state of variables that haven't been assigned a value.

What do you mean by REST API?

A REST API (Representational State Transfer Application Programming Interface) is a set of rules and conventions for building and interacting with web services. It uses standard HTTP methods (like GET, POST, PUT, DELETE) and URLs to allow communication and data exchange between a client (e.g., a web browser or a mobile app) and a server (e.g., a web application or database).
